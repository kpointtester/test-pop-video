Feature 1: Tag-based Embed Loading (player-silk.js / videofront-vega.js)

How it works: Embed div carries data-kvideo-tags instead of data-kvideo-id. The embed script fires a search XHR (/api/v3/search?type=videos&facet.tag=...), then client-side exact-matches the video tag set against the embed tag set (sorted, case-folded). On match it injects the player iframe; on no-match the container stays blank.

Feature 2: Bandwidth-based Resolution (Init and Runtime)

How it works: data-video-params={"bandwidth":"N"} on the embed div. The embed script converts the bandwidth value to a res param and appends it to the iframe URL. HLS.js then starts at the matching level. Runtime: kPlayer1.setBandwidth(N) updates the cap mid-playback.

Known Issue

kplayer.min.js - Cannot set properties of null (setting nextLevel)

Uncaught TypeError: Cannot set properties of null (setting 'nextLevel')
KPlayer.loadMedia @ kplayer.min.js:1

Trigger: Occurs intermittently during player.loadMedia() when this.hls is null at the moment the init-level is being set.
Impact: Playback fails to start in some automated/CDP-driven invocations. In real-browser (user-gesture) flow, the timing is different and the error does not prevent playback.
Recommendation: Add a null-guard in loadMedia() before setting this.hls.nextLevel.
